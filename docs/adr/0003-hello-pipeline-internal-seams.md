# Hello Pipeline owns its Postgres / Redis / Mongo accessors as internal seams

`/api/hello` reads a Visit Count from Postgres, caches the Greeting in Redis, and appends a Hello Event to Mongo. Until this ADR was written, those three stores lived in three public modules — `VisitsRepo`, `RedisCache`, `HelloEventLog` — each a thin wrapper over its Java client. They had exactly one production caller (`HelloHandler`) and no tests; the deletion test for each was negative (folding the body inline concentrates complexity at one place, not many).

**Decision**: the Hello pipeline is a single deep module (`HelloPipeline`). The three stores are package-private traits inside the `helloPipeline` package, wired by `HelloPipeline.live` against Postgres / Lettuce / Mongo, and injected by `HelloPipeline.from(visits, cache, eventLog)` for tests. There is no public `RedisCache` or `HelloEventLog` layer. The `db/` package keeps `DataSource` and `Migrations` because those are infrastructure shared with future, non-Hello pipelines; the `cache/` and `eventlog/` packages are removed.

This is the LANGUAGE.md "internal seams" pattern: the seams that production callers see are the `HelloPipeline.greet` / `recent` / `health` interface; the seams *behind* that interface (`Visits` / `GreetingCache` / `EventLog`) are private and exist only to keep tests sharp. Each has two adapters — live + fake — so by LANGUAGE.md's "two adapters means a real seam" rule they are real internal seams, not hypothetical ones.

The trade is **discoverability of cache / event-log primitives across future pipelines** for **locality of the Hello pipeline as a deep module**. New pipelines that want their own Redis namespace or Mongo collection should introduce their own accessors, not reach into HelloPipeline's. Sharing infrastructure code between pipelines was a future-pulled-forward concern that wasn't paying its keep at the time of writing.

If a second pipeline ever needs the same Redis or Mongo client lifecycle, the right move is to extract the **client lifecycle** (the `RedisClient` + connection acquire-release; the `MongoClient` + index creation) into a shared layer, not the domain accessors. That refactor is not pre-emptively done here.
